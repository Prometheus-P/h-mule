
# 🗺 ROADMAP – Human Mule

> MVP → 파일럿 → 수익 실험까지 한 번에 바라보는 대략의 로드맵.

---

## Phase 0 – PoC (Low-Code) — 약 2~4주

- [ ] 카카오톡 채널 + 간단 웹 폼으로 주문/매칭 PoC
- [ ] 강남 ↔ 여의도 구간 실제 3~5건 테스트
- [ ] 화주 3곳 이상 인터뷰 (단가/느낌/문제)
- [ ] "이게 진짜 돈이 되겠냐?"를 검증하는 단계

---

## Phase 1 – MVP (Go API + Flutter App) — 약 6~8주

- [ ] Go backend 기본 뼈대 (auth, company, messenger, delivery)
- [ ] Flutter 앱:
  - [ ] Company용: 배송 요청 생성/조회
  - [ ] Messenger용: 매칭 수락/진행/완료
- [ ] Rule-based 매칭 (출발역/도착역 + 시간대)
- [ ] 상태 전환(PICKED/DROPPED) + EventLog 기록
- [ ] 최소한의 보안(HTTPS, JWT, 비밀번호 해시)

---

## Phase 2 – B2B 파일럿 — 약 2~3개월

- [ ] 강남/여의도/종로 일부 기업과 Closed Beta
- [ ] B2B 구독 플랜(월 30/50/100건) 실험
- [ ] 정산/리포트 화면 추가
- [ ] CS/분쟁 처리 플로우 정리
- [ ] “서비스를 계속 굴려도 되겠다” 수준의 품질 달성

---

## Phase 3 – 최적화 & 확장

- [ ] 매칭 알고리즘 고도화 (Messenger 평판, 히스토리 기반)
- [ ] Messenger 리텐션(레벨, 뱃지, 보너스 구조)
- [ ] 운영 대시보드: SLA, 실패율, 평균 배송 시간 모니터링
- [ ] 서울 다른 라인 / 수도권 전체 확장 검토

---

## Phase 4 – 고도화 / 새로운 BM

- [ ] 철도공사/지하철 사업자와 제휴 검토
- [ ] 상품 카테고리 확장 (인쇄물/샘플 중심)
- [ ] 관련 B2B 서비스와 번들링(인쇄/로펌/컨설팅 등)

---

## 관리 원칙

- 이 문서는 “살아있는 문서”로, 실제 진행 상황에 따라 수시로 업데이트한다.
- 각 Phase별로 별도의 Milestone을 GitHub에 만들고, Issue를 연결한다.
